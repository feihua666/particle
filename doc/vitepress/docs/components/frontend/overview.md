# 组件总览

Particle 是一个高度模块化的框架，由一系列可重用的组件构成。这些组件分为两大类：全局组件和业务组件。

## 组件架构

Particle 的组件架构遵循以下原则：

1. **高内聚低耦合**：每个组件专注于单一功能，组件间依赖关系清晰
2. **可插拔性**：组件可以按需引入，不影响其他组件的正常运行
3. **标准化接口**：组件间通过标准接口进行通信，便于替换和扩展
4. **开箱即用**：大多数组件提供默认实现，可直接使用

## 全局组件

全局组件是 Particle 的技术基础，提供各种通用技术能力

## 业务组件

业务组件是 Particle 的核心价值所在，提供常见的业务功能

## 组件依赖关系

```
业务组件 -----------┐
    ↓ 依赖         |  
公共模块 (common)   |
    ↓ 依赖         |
全局组件 (global) ← ┘
```

业务组件既可以直接依赖全局组件以获得特定技术能力，也可以通过公共模块间接依赖全局组件。这种灵活的依赖关系使业务组件能够根据具体需求选择最合适的依赖方式。

## 使用建议

1. **按需引入**：根据项目需求选择需要的组件，避免引入不必要的依赖
2. **理解依赖**：在引入组件前，了解其依赖关系，确保依赖组件也已引入
3. **遵循规范**：使用组件时遵循其设计规范和使用约定
4. **扩展定制**：在需要定制功能时，通过扩展而非修改的方式进行

通过合理使用这些组件，您可以快速构建功能丰富、结构清晰的企业级应用。
