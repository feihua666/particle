# 前端全局相关

前端通用的组件或工具应该放在该目录下，该目录下的组件只涉及技术本身，与具体业务无关。这些组件可以在项目的任何地方复用，确保技术栈的一致性和系统的稳定性。

## 目录结构

- [pc](./pc/README.md) - PC端通用技术组件
  - [common](./pc/common/README.md) - 通用工具组件
  - [element-plus](./pc/element-plus/README.md) - Element Plus组件封装

- uniapp - Uniapp移动端通用技术组件

## 核心组件

全局模块是 Particle 前端项目的基础技术支撑层，提供各种通用技术和业务能力的支持。这些模块可以在整个系统中被复用，确保技术栈的一致性和系统的稳定性。

## 设计理念

全局模块的设计遵循以下原则：

1. **高内聚低耦合**: 每个模块专注于特定的技术领域，减少模块间的依赖
2. **开箱即用**: 大多数模块提供合理的默认配置，降低使用门槛
3. **可扩展性**: 模块设计考虑了可扩展性，允许根据具体需求进行定制
4. **统一规范**: 提供系统级的技术规范和标准，确保整个系统的一致性

## 使用说明

全局模块中的各个组件可以通过相对路径引入到项目中，大多数组件都提供了合理的默认配置，可以根据项目需要进行自定义配置。
